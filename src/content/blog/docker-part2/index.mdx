# 1. Run a New Container

- syntax of the command
```bash
docker run [OPTIONS] IMAGE [COMMAND] [ARG...]
```

### Common Flags:
- `-d` ‚Üí Detached mode (run in background)
- `-it` ‚Üí Interactive + TTY (useful for shell access)
- `--name NAME` ‚Üí Assign a custom name
- `-p HOST:CONTAINER` ‚Üí Map host port to container port
- `-P` ‚Üí Publish all exposed ports to random host ports
- `-e KEY=VALUE` ‚Üí Set environment variables
- `--env-file FILE` ‚Üí Load env vars from file
- `-v HOST:CONTAINER` ‚Üí Mount volume
- `--rm` ‚Üí Automatically remove container when stopped
- `--network NETWORK` ‚Üí Connect to specific network
- `--restart [no|always|on-failure:n|max-retries]` ‚Üí Restart policy
- `--hostname NAME` ‚Üí Set container hostname
- `--entrypoint CMD` ‚Üí Override image entrypoint
- `--cpus N` ‚Üí Limit CPU usage
- `--memory SIZE` ‚Üí Limit RAM usage
- `--user [USER, UID, UID:GID]` ‚Üí Run container as specific user

Example:
```bash
docker run -dit --name web -p 8080:80 -v $(pwd):/usr/share/nginx/html nginx
```

# 2. List Containers
```bash
docker ps [OPTIONS]
```
### Flags:

- `-a` ‚Üí Show all containers (default shows only running)
- `-q` ‚Üí Show only container IDs

### 3. Start / Stop / Restart / Pause / Unpause Containers

- `pause` suspends all processes inside a container
```bash
docker start CONTAINER
docker stop CONTAINER
docker restart CONTAINER
docker pause CONTAINER
docker unpause CONTAINER
```

### 4. Execute a Command Inside a Running Container
```bash
docker exec [OPTIONS] CONTAINER COMMAND
```
Flags:
- `-it` ‚Üí Interactive terminal
- `--user USER` ‚Üí Run command as specific user
- `-w DIR` ‚Üí Working directory inside container
Example:
```bash
docker exec -it web bash
```

### 5. View Container Logs
```bash
docker logs [OPTIONS] CONTAINER
```
Flags:
- `-f` ‚Üí Follow logs (like tail -f)
- `--tail N` ‚Üí Show last N lines
- `--since TIME` ‚Üí Show logs since specific time
- `--until TIME` ‚Üí Show logs until specific time

### 6. Inspect a Container
```bash
docker inspect CONTAINER
```
Flags:
- `--format '{{.State.Status}}'` ‚Üí Show specific field
- `docker inspect --format '{{ .NetworkSettings.IPAddress }}' myapp`
- `docker inspect --format '{{ .Config.Image }}' myapp`


### 7. Copy Files To/From container
```bash
docker cp CONTAINER:/path/to/file host_path
docker cp host_path CONTAINER:/path/to/file
```
`
### 8. Rename/Remove a Container

- Rename container
```bash
docker rename OLD_NAME NEW_NAME
```
- Remove Containers
```bash
docker rm [OPTIONS] CONTAINER
```
- Flags:
- `-f` ‚Üí Force removal (kill if running)
- `-v` ‚Üí Remove anonymous volumes attached

### 9. Extra
- To remove all stopped containers: `docker rm $(docker ps -aq)`
- To stop all running containers: `docker stop $(docker ps -q)`
- To clean up everything (containers, images, networks, volumes): `docker system prune -a --volumes`
